# PR 코드 리뷰

다음 관점에서 현재 변경된 코드를 체계적으로 리뷰해줘:

## 리뷰 범위

1. **보안** (OWASP Top 10 기준)
   - 인젝션 취약점
   - 민감 데이터 노출
   - 인증/인가 로직

2. **코드 품질**
   - 네이밍 명확성
   - 중복 코드 (DRY 원칙)
   - 단일 책임 원칙

3. **성능**
   - N+1 쿼리
   - 불필요한 연산
   - 메모리 누수 가능성

4. **라이브러리 사용**
   - Deprecated API 여부
   - 최신 권장 패턴

## 출력 형식

```
🔴 Critical: [즉시 수정 필요한 항목]
🟡 Warning:  [수정 권장 항목]
🟢 Suggestion: [개선 제안]
✅ Good: [잘 작성된 부분]
```
