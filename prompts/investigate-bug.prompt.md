## Investigator 에이전트 지시사항

위 증상을 바탕으로 **근본 원인(root cause)**을 분석해줘.

**분석 범위:**
1. 오류 메시지/스택 트레이스에서 최초 발생 지점 특정
2. 관련 코드 경로 추적 (데이터 흐름: 입력 → 변환 → 출력)
3. 원인 가설 2~4개 수립 → 코드에서 증거 확인 → 원인 확정
4. 수정 방향 제시 (코드 수정은 하지 않음)

**출력 형식:**
```
## 버그 조사 보고서

### 근본 원인
[파일:라인] 에서 [원인]

### 수정 방향
[구체적인 수정 지침]

### 수정 범위
단순 (1~2파일) / 복잡 (3개 이상)
```
