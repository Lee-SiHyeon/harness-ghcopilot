# 하네스 출시 체크리스트

출시 전 아래 항목을 **순서대로** 점검해줘. 각 항목에 ✅/❌/⚠️ 표시해줘.

---

## 1. 보안 (OWASP Top 10)

- [ ] 하드코딩된 시크릿(API 키, 비밀번호, 토큰) 없음
- [ ] 사용자 입력에 인젝션 취약점 없음 (SQL, Command, LDAP)
- [ ] 인증·인가 로직 올바르게 적용됨
- [ ] 민감 데이터가 로그에 기록되지 않음
- [ ] 의존성 패키지 알려진 취약점 없음 (`npm audit` / `pip audit`)

## 2. 에이전트 하네스 품질

- [ ] 모든 에이전트가 `tools` 최소 권한만 보유 (불필요한 `edit`, `execute` 없음)
- [ ] `read-only` 에이전트(Planner, Reviewer, Investigator)가 파일을 수정하지 않음
- [ ] Maestro 파이프라인 분류 표가 최신 에이전트 목록과 일치함
- [ ] 버그 수정 플로우에 Investigator 단계가 포함됨
- [ ] 훅 스크립트 타임아웃이 적절함 (5초 이내)

## 3. 코드 품질

- [ ] 타입스크립트 컴파일 오류 없음 (`tsc --noEmit`)
- [ ] 린트 경고 없음 (`eslint` / `ruff`)
- [ ] 테스트 전체 통과 (`npm test` / `pytest`)
- [ ] 중복 코드 없음 (DRY)
- [ ] 함수/변수명 명확함

## 4. 관찰성 (Observability)

- [ ] 에이전트 파이프라인 실행이 로그에 기록됨
- [ ] 오류 발생 시 원인 파악 가능한 메시지 출력됨
- [ ] 훅 실패 시 `continue: true` 폴백이 있어 전체 중단 없음

## 5. 테스트 커버리지

- [ ] 핵심 경로(happy path) 테스트 존재
- [ ] 엣지 케이스 처리 확인
- [ ] 통합 테스트 (여러 에이전트 협업 시나리오) 검증

## 6. 문서화

- [ ] README 최신 상태
- [ ] 새로 추가된 에이전트/스킬에 description 작성됨
- [ ] `.env.example` 환경변수 목록 최신화

---

## 체크리스트 실행 지침

위 항목을 **Reviewer 에이전트**에게 위임해서 실제 코드를 점검하도록 한다.
Critical 항목(보안 + 하네스 품질) 불합격 시 출시 중단.
