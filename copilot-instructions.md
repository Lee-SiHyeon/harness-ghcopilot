# GitHub Copilot Repository Instructions

## Context7 — 공식 문서 기반 코드 생성

라이브러리, 프레임워크, SDK, API, CLI 도구에 관한 질문이 있을 때는 **항상 Context7 MCP 도구**를 사용하여 최신 공식 문서를 가져온 뒤 답변한다.

### Context7 사용 규칙

1. **라이브러리 관련 코드 작성 전** 반드시 Context7로 최신 문서를 확인한다.
2. `resolve-library-id` 도구로 라이브러리 ID를 먼저 확인한다.
3. `query-docs` 도구로 해당 기능의 공식 문서 스니펫을 가져온다.
4. 학습 데이터(training data)만으로 라이브러리 API를 추측하지 않는다.
5. 사용자가 특정 버전을 언급하면 그 버전의 문서를 Context7에서 조회한다.

### 적용 범위

- 패키지 설치 방법, 설정(configuration) 코드 작성
- API 메서드·옵션·파라미터 확인
- 프레임워크 초기화 및 boilerplate 생성
- 버전 마이그레이션 가이드
- 예시: React, Next.js, Prisma, Express, Tailwind, Django, Spring Boot 등 모든 외부 라이브러리

### 예시 프롬프트 패턴

```
Next.js 14 미들웨어 설정해줘  → Context7로 Next.js 14 문서 조회 후 작성
Supabase 이메일 로그인 구현  → Context7로 Supabase auth 문서 조회 후 작성
```

## Maestro — 복잡한 작업의 오케스트레이션 패턴

Agent 모드에서 복잡한 요청을 받으면 **Maestro 오케스트레이션 패턴**으로 처리한다.

### 복잡한 요청 판단 기준
- 3개 이상 파일 수정이 예상될 때
- 신규 기능 설계 + 구현이 함께 필요할 때
- 보안/품질 검토가 필요한 코드 변경일 때
- 문서화 + 코드 생성이 함께 필요할 때

### 오케스트레이션 실행 순서
1. **분류**: 요청 유형 결정 (신규기능 / 버그수정 / 리팩토링 / 문서화 / 리뷰)
2. **탐색**: 코드베이스 파악 + Context7 라이브러리 조회 (병렬)
3. **계획**: Planner 서브에이전트 호출 → 구현 계획서 생성
4. **구현**: Implementer 서브에이전트 호출 → 계획서 기반 코드 작성
5. **검증**: Reviewer 서브에이전트 호출 → OWASP 보안·품질 검토
6. **반복**: 이슈 발견 시 4→5 최대 3회 순환

> 단순 질문, 단일 파일 수정, 빠른 조회는 직접 처리한다. @Maestro는 멀티-에이전트 협업이 필요한 작업의 명시적 진입점이다.

## 일반 코딩 지침

- 코드는 간결하고 관용적으로(idiomatic) 작성한다.
- 불필요한 주석, 과도한 에러 핸들링, 불필요한 추상화를 추가하지 않는다.
- 요청된 변경 사항만 수정하고, 관련 없는 코드는 건드리지 않는다.
